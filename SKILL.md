# 哲学论文学术报告 Handout 制作技能

本技能指导如何从一篇哲学论文原文出发，制作出符合分析哲学规范的高质量学术报告 handout。输出为 **LaTeX 源文件**（.tex），可编译为专业排版的 PDF。

## 核心理念与排版风格

### Markdown极简风格

本模板采用**类Markdown的极简排版**风格：

1. **无装饰**：无任何彩色框、线框、背景色块
2. **纯文字层次**：仅通过标题、列表、表格、公式组织内容
3. **高信息密度**：每页包含尽可能多的关键信息
4. **黑白打印友好**：纯黑色文字，无灰度依赖

### 双独立版本结构

Handout 采用**两份完全独立的handout**结构：
- **英文完整版**：第一遍完整呈现所有内容
- **中文完整版**：第二遍完整翻译，非摘要

两份handout结构平行、内容对应，而非混合排版。

## 输出格式：LaTeX

**必须使用 LaTeX 编写 handout**，理由：
- LaTeX 天然支持数学公式与逻辑符号（$\forall, \exists, \rightarrow, \models$）
- 精确控制排版，实现极简高密度布局
- TikZ 绘制论证图
- booktabs 提供无竖线表格（符合极简美学）
- ctex 宏包完美支持中英混排

请先阅读 `references/latex-template.tex` 获取完整的 LaTeX 模板，然后基于该模板编写 handout。

## 全流程方法论：七阶段工作流

### 阶段一：文本解析与论点提取

通读论文全文（至少两遍），执行以下操作：

**第一遍——宏观结构扫描：**
- 识别论文的章节划分与逻辑顺序
- 定位核心论题（thesis）：通常出现在引言最后一段或结论段
- 识别理论对手（theoretical opponent）：作者要反驳或修正的立场
- 标注所有思想实验（thought experiments）的名称、位置和目的

**第二遍——微观论证拆解：**
- 逐节提取：每节的主张（claim）、前提（premises）、推理模式（inference pattern）
- 识别论证类型：演绎（deductive）、归纳（inductive）、溯因（abductive）、归谬（reductio）
- 标注所有反驳-回应对（objection-reply pairs）
- 记录关键定义出现的页码/段落

**输出物**：一份结构化的论文分析笔记，包含：
- 核心论题的一句话表述
- 理论对手的一句概括
- 逐节论证摘要（每节 2-3 句话）
- 关键概念清单（术语 + 文中定义 + 页码）
- 思想实验索引表
- 反驳-回应索引表

### 阶段二：概念定义清单编制

列出论文中所有需要明确定义的关键术语。每个概念条目包含：

| 字段 | 说明 |
|------|------|
| 术语 | 原文使用的术语（保留英文原文） |
| 定义 | 作者给出的精确定义（中英双语） |
| 首次出现位置 | 页码或章节号 |
| 与其他概念的关系 | 如适用 |

**关键原则**：
- 保留原文术语的英文形式，用 `\term{}` 命令标注
- 区分作者的专门用法与日常用法
- 注意同一概念在不同段落中可能出现的定义变化

### 阶段三：论证结构形式化

这是本技能最具特色的核心步骤。将论文中的关键论证转化为形式化表达。

**论证-schema 格式：**

```latex
\textbf{P1:} Premise one content \pagemark{page}\\
\textbf{P2:} Premise two content \pagemark{page}\\
\textbf{C:} Conclusion content \pagemark{page}
```

**形式逻辑表达：**

当论证可以被逻辑形式化时，使用以下符号体系：
- 命题逻辑：$\neg$（否定）, $\land$（合取）, $\lor$（析取）, $\rightarrow$（实质蕴涵）, $\leftrightarrow$（等价）, $\models$（逻辑后承）
- 一阶逻辑：$\forall$（全称量词）, $\exists$（存在量词）
- 模态逻辑（如适用）：$\Box$（必然）, $\Diamond$（可能）
- 认知逻辑（如适用）：$K_a \varphi$（a知道φ）, $B_a \varphi$（a相信φ）

**示例**（Dennett 的真信徒准则）：
```latex
设 $I(x)$: $x$ 是意向系统，$P(x)$: $x$ 的行为可通过意向策略可靠预测。

核心准则：
\[ \forall x \, (I(x) \leftrightarrow P(x)) \]
```

**分类体系的形式化：**

当论文涉及概念分类时，使用结构化表示：
```latex
实践推理的类型：
\begin{enumerate}[label=(\greek*)]
  \item \textbf{审慎推理}（Prudential reasoning）：以欲望为驱动
  \item \textbf{制度推理}（Institutional reasoning）：以社会身份为驱动
  \item \textbf{无条件推理}（Unconditional reasoning）：与欲求和身份均无关
\end{enumerate}
```

### 阶段四：Handout 宏观结构设计

采用**双独立版本**结构组织 handout 内容：

**英文完整版结构：**
| # | 模块 | 内容 |
|---|------|------|
| 1 | Header | 论文标题、作者、汇报人、日期 |
| 2 | Thesis | 一句话核心主张 |
| 3 | Background | 理论背景、被挑战的假设 |
| 4 | Arguments | 核心论证详细展开 |
| 5 | Definitions | 关键概念精确定义 |
| 6 | Objections | 反驳与回应 |
| 7 | Commentary | 汇报者评论 |

**中文完整版结构：** 与英文版结构完全平行，完整翻译所有内容。

### 阶段五：LaTeX 编写实施

**必须基于 `references/latex-template.tex` 模板编写**，该模板预配置了：
- Markdown极简风格（无框线、无颜色块）
- 高信息密度排版设置
- 所有必要的宏包（amsmath, amssymb, booktabs, tikz, ctex 等）
- 专业排版设置（紧凑字体、间距）

**编写时遵循以下规则：**

1. **无装饰原则**：不使用任何彩色框、线框环境。使用纯文本、列表、表格组织内容。

2. **标题层级**：
   - `\section*{}` 用于主要章节
   - `\subsection*{}` 用于子节
   - `\paragraph{}` 用于更细的条目

3. **强调方式**：仅使用粗体(`\textbf{}`)和下划线(`\underline{}`)，不使用颜色。

4. **表格使用**：仅使用三线表（`\toprule`, `\midrule`, `\bottomrule`），无竖线。

5. **页码引用**：使用 `\pagemark{页码}` 标注原文出处。

6. **术语标注**：使用 `\term{English Term}` 标注英文术语。

### 阶段六：论证图绘制（Argument Map）

对于核心论证，使用 TikZ 绘制 **论证结构图**，置于 handout 末尾或相关章节之后。

**论证图规范（极简风格）：**
- 使用简单矩形表示前提和结论
- 使用实线箭头表示推理方向
- 不使用颜色、圆角、阴影等装饰
- 紧凑布局，最大化信息密度

## 写作风格规范

### 分析哲学三大原则

1. **清晰性（Clarity）**：每句一个主谓结构，拒绝模糊指代。使用限定词（"某些情况下""原则上""在……条件下"）避免过度概括。

2. **简洁性（Conciseness）**：删除冗余副词和重复表述。用短句替代长复合句。避免"非常""极其""十分"等程度副词。

3. **精确性（Precision）**：严格区分近义哲学术语（如"信念"vs"知识"vs"意见"）。首次使用关键术语时即给出定义。使用形式化手段消除自然语言的歧义。

### 引用与评注分离

这是 handout 写作中最重要的规范之一：
- **原作内容**：忠实转述作者观点，标注页码
- **汇报者评论**：使用显式标记（"笔者认为""值得注意的是"）
- 严禁将个人观点混入对原文的转述中

### 页码引用规范

使用 `\pagemark{页码}` 命令标注原文出处。格式示例：
- "布兰顿提出了实质推论的概念\pagemark{87}。"
- 当引用特定段落时："\pagemark{§3, 89}"（第3节，第89页）
- 同时标注中英版页码时："\pagemark{英89/中79}"
